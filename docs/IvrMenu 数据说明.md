# IvrMenu 数据说明

当服务器需要通知GUI客户端显示 IVR 菜单时，它发送 24 号消息，消息的整数部分是 0，字符串部分是 JSON 定义。

定义举例:

```javascript
[
    {
        // 菜单标题，必填
        "caption": "主菜单 1",

        // 菜单说明，可以省略，表示没有说明
        "description": "向用户播放我们的公告",

        // 传递给 CTI 服务器的参数
        // args 与 children 不能共存，亦即有子菜单的，就没有实际的调用动作
        // "args": { ... }

        // 子菜单
        "children": [
            "caption": "子菜单 1.1",

            // 技能组ID数组，只有签入这些技能组后才能使用；如果为 null(默认值) 或者空数组，就表示没有签入数组的限制
            "groups": ["一组", "二组"],

            // 客户端按照这个调用 CTI 的 IVR 流程
            "args": {
                "ivrId": "ivr-1",  // 要使用的 IVR 的流程ID
                "callType": 0,  // 0 | 1，必须
                "params": ["foo", "bar"], // 如果增加有Param参数（自定义参数串），则自行增加对应的变量
            }
        ],
    },
    {
        // 菜单标题，必填
        "caption": "主菜单 2",
        "args": {
            "ivrId": "ivr-2",  // 要使用的 IVR 的流程ID
            "callType": 1,  // 0 | 1，必须
            "params": [], // 如果增加有Param参数（自定义参数串），则自行增加对应的变量
        }
    }
]
```
